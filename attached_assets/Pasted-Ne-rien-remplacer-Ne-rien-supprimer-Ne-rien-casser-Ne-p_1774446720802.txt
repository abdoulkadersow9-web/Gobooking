3. RÔLES À UTILISER (PROGRAMME RÉEL)

Organiser les agents selon ces rôles principaux :

- Agent en route
- Agent guichet
- Agent colis (incluant réception)
- Agent embarquement
- Agent logistique (si existant)

Supprimer les incohérences d’affichage (sans supprimer les données), et regrouper correctement.

---

4. AGENT EN ROUTE

- lié à un départ
- 1 départ = 1 agent en route
- afficher clairement :
  - nom
  - téléphone
  - départ
  - statut

---

5. AGENT GUICHET

- lié à une agence
- gestion des ventes physiques

---

6. AGENT COLIS (TRÈS IMPORTANT)

Fusionner la logique :

- Agent réception = Agent colis

Ne plus séparer ces deux rôles.

---

7. MODULE COLIS (À ADAPTER)

Dans l’espace Agent Colis :

Garder le système actuel mais organiser avec deux onglets :

A. Onglet "Créer colis"
- création colis
- génération code colis

B. Onglet "Retrait colis"
- saisie du code colis
- saisie du mot de passe / code de retrait
- validation du retrait
- confirmation colis livré

---

8. LOGIQUE RETRAIT

Quand le client vient retirer :

- l’agent entre le code colis
- demande le code de retrait (mot de passe)
- vérifie
- valide le retrait

Après validation :
- statut = livré / retiré
- mise à jour automatique
