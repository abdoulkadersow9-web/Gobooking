🔴 2. STRUCTURE MÉTIER À RESPECTER (TRÈS IMPORTANT)

Organisation stricte :

- Une COMPAGNIE possède plusieurs AGENCES
- Chaque AGENCE possède ses AGENTS :
  - agent guichet
  - agent bagage
  - agent embarquement
  - agent colis
  - agent en route
- Exception : agent réservation → voit toutes les réservations de la compagnie

👉 Cette hiérarchie doit être respectée partout dans l’application sans mélange.

---

🔴 3. REMONTÉE DES INFORMATIONS (LOGIQUE GLOBALE)

Toutes les actions doivent remonter automatiquement :

- ventes tickets
- bagages
- colis
- embarquement
- validations

👉 Tout doit être visible :

- au chef d’agence
- à l’entreprise (compagnie)

👉 Aucune donnée ne doit rester isolée.

---

🔴 4. LOGIQUE DES BORDEREAUX (FLUX COMPLET)

Flux attendu :

1. Les agents effectuent leurs opérations (vente, bagage, embarquement)
2. Les données remontent automatiquement
3. L’agent de validation valide le départ

👉 Dès validation :

- le bordereau initial est généré
- il est envoyé automatiquement :
  - au chef d’agence
  - à l’entreprise

👉 À ce niveau :

- la compagnie voit déjà la performance du départ (recettes, activité)
- même si toutes les dépenses ne sont pas encore ajoutées

---

🔴 5. AJOUT DU BORDEREAU CARBURANT (LOGIQUE AVANCÉE)

Le bordereau carburant peut arriver plus tard (le même jour ou après).

👉 Donc :

- le chef d’agence doit voir une liste des départs déjà effectués
- avec un indicateur clair :
  - “bordereau carburant manquant”

👉 Action chef d’agence :

- il sélectionne un départ
- il ajoute (scan ou saisie) le bordereau carburant
- il valide

👉 Résultat :

- les données du départ sont mises à jour :
  - recettes tickets
  - bagages
  - colis
  - + carburant
- un bordereau final COMPLET est généré

👉 Et automatiquement :

- les données mises à jour sont renvoyées à la compagnie

---

🔴 6. NIVEAU ENTREPRISE (COMPAGNIE)

La compagnie doit voir :

- tous les départs
- les recettes (immédiates après validation)
- les dépenses (après ajout carburant)
- les résultats nets mis à jour

👉 Important :

- avant carburant → performance brute
- après carburant → résultat réel net

👉 Objectif :

- calcul propre
- prêt pour comptabilité
- aucune manipulation manuelle
