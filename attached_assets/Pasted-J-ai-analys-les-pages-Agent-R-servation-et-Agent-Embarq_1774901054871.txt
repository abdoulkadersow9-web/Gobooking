## 1. PROBLÈME – PAGE AGENT RÉSERVATION

Actuellement :

- toutes les réservations sont mélangées  
- aucune distinction claire entre :
  - réservations des bus déjà partis (en route)
  - réservations des départs futurs  

👉 Cela crée :
- confusion  
- erreurs possibles  
- mauvaise expérience agent  

---

## 2. LOGIQUE À APPLIQUER (OBLIGATOIRE)

Il faut séparer clairement :

### A. RÉSERVATIONS – BUS EN ROUTE

- bus déjà parti  
- places restantes disponibles  
- réservations en temps réel  

👉 rôle agent :
- confirmer les réservations immédiatement  
- gérer les places restantes  

---

### B. RÉSERVATIONS – DÉPARTS FUTURS

- bus pas encore parti  
- réservations normales  

👉 rôle agent :
- valider / confirmer les réservations  

---

## 3. STRUCTURE UI À FAIRE

Créer une interface moderne avec séparation claire :

- onglet ou filtre :
  - En route
  - À venir

OU

- deux blocs visuels distincts :
  - réservations en cours (priorité)
  - réservations futures  

👉 priorité visuelle sur "En route"

## 4. OBJECTIF

- éviter toute confusion
